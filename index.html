Not quite the same thing


          aria-label

vs.


          aria-labelledby

aria-labelledby


          <input aria-labelledby="label1">
          
          <p id="label1">Some label<p>
  • Requires a separate element's ID
  • Useful if content already exists on the page
  • Can be automatically translated

aria-label


          <input aria-label="Some label">
  • Requires a text string
  • Useful if content not already on the page
  • Cannot be automatically translated

aria-describedby


          <input aria-describedby="SomeID">
          <p id="SomeID">Some text describing the input<p>
  • Requires a separate element's ID
  • Useful to associate elements together programmatically

aria-label

Cannot be automatically translated!

<label>

If there's any chance of a user needing translation, the label element is better

Might or might not work


          <div aria-describedby="randomid1">
          
          <p aria-labelledby="randomid2">
          
          <span aria-label="Some random stuff">

Not quite the same thing


          aria-hidden

vs.


          role="presentation"

aria-hidden

  • Hides the element and its content from assistive technology

role="presentation"

  • Removes the semantic meaning of the element

Error messages - Found in the wild


          <input type="text" id="FirstName">
          <p>First name can't be empty<p>
  • No association between input and error message.

Error messages - A better approach


          <input type="text" id="FirstName" aria-describedby="errFirstName">
          <p id="errFirstName">First name can't be empty<p>
  • Explicit association of the two elements.

Required fields - Visual notification


          <label for="FirstName">First Name *</label>
  • Asterisk announced as "star". May or may not be easily understood by screen reader users
  • Adds to the cognitive load

Required fields - A better way


          <label for="FirstName">First Name *</label>
  • Asterisk announced as "star". May or may not be easily understood by screen reader users
  • Adds to the cognitive load

Required fields - Programmatic notification


          <label for="FirstName">First Name <span aria-hidden="true">*</span></label>
  • Programmatically hide the asterisk for screen reader users
  • Reduce cognitive load
  • Assumes the <input> has been programmatically amd correctly marked as required.

Required fields


          <input required type="text">

vs.


          <input aria-required="true" type="text">

Required fields - required HTML5 attribute

Don't use ARIA if there's a native HTML way of doing things, right?

  • Automatically announces a field as “invalid” the 1st time focus is set to it
  • Triggers a client-side validation that fails WCAG.
  • Generic, non persistent error messages.

Required fields - aria-required ARIA attribute

  • Does not suffer from the downfalls of the "required" attribute!
